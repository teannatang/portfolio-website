Study Buddies

Since this project is still being worked on, not much of the prototyping has been completed yet. However, once the project gets closer to completion, more information will be continuously added.

Studying can be difficult for many people, making it hard for students to stay focused and ace exams. However, Study Buddies provides various studying methods that implement a variety of games, achievements, and study tools that help make it easier to stay engaged for a longer period of time.

My Role: UX/UI Design, User Research, and User Testing
Duration: Aug 2021 – current

Research

User Story: Julius Masangkay (24)

As a busy user traveling to Mexico soon, I want to learn basic Spanish vocabulary and phrases to create more meaningful interactions during my stay.

Challenges:

  • Has limited amount of time due to frequent local and international travels to teach at various dance events
  • Works full-time as a full-stack developer
  • Needs an app with fun learning tools to stay engaged, or will lose interest and become unmotivated to study

Ideation

Task Flow #1: Add a New Flashcard

  1. Entry Point: Home Screen
  2. Select "New Flashcard"
  3. Select Category
  4. New Set/New Folder
  5. Tap "Title"
  6. Type Title in Textbox
  7. Tap "Description"
  8. Type Description in Textbox
  9. Tap "Term"
  10. Type Flashcard Term in Textbox
  11. Tap "Definition"
  12. Type Flashcard Definition in Textbox
  13. Select Category: "Add to Folder?"
  14. Yes/No
  15. Success Criteria: Begin Learning

Task Flow #2: Play Learning Game

  1. Entry Point: Home Screen
  2. Select "Play"
  3. Choose Desired Set: Set "X"
  4. Select Category: Matching/Basketball/Zombies
  5. Game Instruction Screen
  6. Choose Level: Easy/Medium/Hard
  7. Play Game
  8. Success Criteria: Final Score Appears

User Testing

In order to see how other users would interact with the app and if the user flows were intuitive or not, I conducted user testing on 3 participants who were given the following tasks:

  1. Create a New Study Buddies Account
  2. Add a New Buddy
  3. Add a New Flashcard to an Existing Set
  4. Play the Basketball Learning Game

Findings

With Usability Testing, I was able to identify 2 challenges that the participants experienced while interacting with the app.

Issue #1: Users were slightly confused on what to expect from the titles of the main categories on the home page during Task #3.


  • Improvement: Rename the "Create" category to "Add New Flashcard" and "Flashcards" rename to "My Study Sets" to make it more clear on what to expect when interacting with the categories.

Issue #2: Users were unable to find the game feature and didn’t know it even existed until further exploration during Task #4


  • Improvement: Add a “Play” Category to make it more known to users that the feature exists when opening to the home page

Next Steps

  • Add a Home Button for users to go straight to the home page instead of having to click the “Back” button multiple times
  • Create onboarding screens for new Home Screen
  • Create wireframe prototypes for the updated flow of the app
  • Create a high-fidelity prototype on AdobeXD
  • Conduct more User Testing for each updated version

Final Takeaways

Pros

  • Learn to ask “Why?” to ensure opinions are unbiased during user testing
  • Your flow may make sense to you but not to other people, which is why user testing is so important
  • Take all feedback into consideration, even if they seem minor
  • Have fun during the process! :)

Cons

  • Don’t procrastinate on projects, it's always better to do a little at a time to prevent burnout
  • New issues come up with every Usability Testing session, which also means the app will need new modifications to the prototype. Therefore, the process of getting to the final version of an app takes time.
