PlutoPay

PlutoPay is a mobile virtual wallet app used to help make transactions easy and more efficient. The app is designed for tech-savvy individuals who want a quicker way to shop or complete transactions and a way to help practice better spending habits.

My Role: UX/UI Design, User Research, and User Testing
Duration: Aug 2021 – Feb 2022

Research

Before going into the design of an app, it is important to first identify the problem in order to create actual solutions. Therefore, I did an extensive amount of research in order to have a good understanding of wireless payments so that users feel safe enough to give PlutoPay their private information.

Task Flow #1: Paying Back a Friend

  1. Entry Point: Home Screen
  2. Transactions
  3. Enter Amount & Description
  4. Press "Pay"
  5. Choose Payment Method
  6. Select User
  7. Press "Send"
  8. Confirm Account/Skip
  9. Press "Done"
  10. Success Criteria: Confirmation Screen Appears "Transaction Complete!"

Task Flow #2: Requesting Money From a Friend

  1. Entry Point: Home Screen
  2. Transactions
  3. Enter Amount & Description
  4. Press "Request"
  5. Select User
  6. Press "Send"
  7. Success Criteria: Confirmation Screen Appears "Transaction Complete!"

Findings

Most of the assigned tasks during user testing were completed successfully by all participants. However, I discovered three issues that would be concerning to the success of the app if they were left unfixed. Therefore, I came up with solutions for these two issues and made the necessary changes on the final prototype. The 3 issues and their corresponding solutions are shown below.


Issue #1: Users were unable to identify all the main functions of the app after interacting with the app.

Improvement: Add onboarding that introduces all main features of the app.

Issue #2: Users struggled to complete the task "Add a New Payment Method and would need assistance.

Improvement: Rename "Wallet" to "My Wallet" in order to make this option more clear.

Issue #3: Users were unable to interpret the shopping icon successfully.

Improvement: Change current shopping icon to a shopping cart because it is more common.

Prototype

Many changes were made and additional testing was conducted prior to creating the high-fidelity prototype for PlutoPay. I conducted preference testing to determine a finalized UI theme as well as made changes to the original user flows where participants struggled to complete during user testing. Through all the trial and error, I quickly realized the importance of conducting these tests because having other perspectives help me find issues that I would not be able to find myself.
